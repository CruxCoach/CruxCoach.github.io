Privacy

How cruxcoach.org handles your data — short version: it doesn't.

What this site does

cruxcoach.org serves static HTML, CSS, SVG, and PNG files. The pages set no cookies, embed no analytics, and load no third-party scripts or fonts. All JavaScript, CSS, and library code that the site uses is served from cruxcoach.org itself — there are no CDN-hosted dependencies.

Two pages contain JavaScript, and only for the feature each one needs:

  • /boards/ renders an interactive map of climbing-board locations. It uses a locally-hosted copy of the open-source Leaflet library, and loads its dataset (/boards/data/boards.geojson) from this site. It also saves your last map position and filter selections in your browser's localStorage so a page reload restores your view — that data stays on your device and is never transmitted. The map tiles, however, are fetched from a third-party tile server — see the next section.
  • /404.html, when reached via a /c/<naddr> URL, opens WebSocket connections to public Nostr relays (relay.damus.io, nos.lol, relay.primal.net) to fetch the metadata for a shared climb. No data is sent except the standard Nostr query.

Map tiles on /boards/

The map background on /boards/ is rendered from OpenStreetMap raster tiles served by tile.openstreetmap.org. While you are viewing that page, your browser sends requests directly to that host for every tile it needs to draw. Those requests are subject to OpenStreetMap's tile-usage policy and the OpenStreetMap Foundation's privacy policy. We have no insight into those requests; the only way to avoid them is to not load the map.

What Codeberg does

This site is hosted by Codeberg e.V. on Codeberg Pages. Codeberg's web server records each request — typically the source IP, timestamp, requested path, response status, and user agent — for operational reasons such as abuse prevention and traffic visibility. We have no access to those logs. See Codeberg's privacy policy for details.

External links

When you click a link to another site (Codeberg.org, Zapstore.dev, Mastodon.social, Primal.net, etc.), your browser connects directly to that operator. Those services have their own privacy practices, which apply at the moment you connect:

  • Codeberg: docs.codeberg.org/improving-codeberg/privacy-policy/
  • Zapstore: zapstore.dev/docs/faq
  • Mastodon (mastodon.social): mastodon.social/privacy-policy
  • Primal: primal.net

Email

The address press@cruxcoach.org is forwarded to the project maintainer. Email content and routing metadata pass through the operator that provides the forwarding service. We retain no separate copy beyond what's needed to reply, and we don't use the address for any newsletter, marketing, or unsolicited contact.

The CruxCoach Android app

This page describes only the cruxcoach.org website. The CruxCoach Android app has its own data-handling model — local-only by default, no telemetry, no accounts, opt-in encrypted backup. The app's network behaviour is described on the main page and in the source repository at codeberg.org/CruxCoach/CruxCoach.
