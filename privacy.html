Privacy

No user tracking, cookies, profiles, or stored visitor/analytics identifiers. Only coarse daily event counters.

What this site does

cruxcoach.org serves static HTML, CSS, SVG, and PNG files. The pages set no cookies, load no third-party analytics, and load no third-party scripts or fonts. All JavaScript, CSS, and library code that the site uses is served from cruxcoach.org itself — there are no CDN-hosted dependencies. The first-party components send only the anonymous aggregate events described below.

Every direct-APK button points to the same first-party selector at stats.cruxcoach.org. A background process reads the release manifest published at /.well-known/apk-target.json, checks Codeberg through its release API and, when Codeberg is not confirmed healthy, checks the byte-identical Zapstore mirror from the server, then caches the working choice. Your browser makes no release/API/HEAD availability request. Only after you activate the one APK button does the selector redirect your browser to the currently selected source.

Some pages contain JavaScript, and only for the feature each one needs:

  • /boards/ renders an interactive map of climbing-board locations. It uses a locally-hosted copy of the open-source Leaflet library, and loads its dataset (/boards/data/boards.geojson) from this site. It also saves your last map position and filter selections in your browser's localStorage so a page reload restores your view — that data stays on your device and is never transmitted. The map tiles, however, are fetched from a third-party tile server — see the next section.
  • /404.html, when reached via a /c/<naddr> URL, opens WebSocket connections to public Nostr relays (relay.damus.io, nos.lol, relay.primal.net) to fetch the metadata for a shared climb. No data is sent except the standard Nostr query.

Anonymous aggregate counters

On a page load, the local /assets/anonymous-analytics.js script sends one first-party counter request to stats.cruxcoach.org; it sends another when you activate the Zapstore install option. A direct-APK activation is counted by the same first-party request that selects and redirects to an available APK source, so it is not sent a second time by JavaScript. If your browser has Do Not Track or Global Privacy Control enabled, the script sends nothing and the selector skips its counter while still performing the download redirect.

  • A page view contains only a canonical page name such as /, /boards/, or /privacy.html. Query strings, fragments, referrers, and full URLs are not sent. Every /c/<naddr> share is reduced to the same /c/:share label; the share identifier never reaches the counter.
  • An install click contains only the target (direct_apk or zapstore), placement (hero, install, or shared_climb), and page language (en/de).
  • The script requests explicitly omit credentials and the referrer; direct-APK links and redirect responses also set a no-referrer policy. No cookie, local-storage ID, account/Nostr key, device ID, fingerprint, screen data, IP hash, or session/visitor identifier is created or transmitted. The selector reads a DNT/GPC header only to suppress counting and does not retain it.

The collector immediately increments an allowlisted UTC-day aggregate and retains no raw event, exact timestamp, request header, IP address, user agent, or referrer. Its own request logging and the dedicated reverse proxy's access logging are disabled. An IP address is necessarily processed transiently by network infrastructure to deliver the request, but it is not copied into the event payload or aggregate record, logged, hashed, or retained by our analytics stack. The resulting numbers are page views and clicks — never people, sessions, or unique visitors. They are approximate: privacy signals, blockers, and outages can reduce them, while reloads, bots, and forged requests can increase them. The collector and dashboard source are published in the cruxcoach-dlstats repository.

Controller, purpose and legal basis

The controller for the first-party analytics processing at cruxcoach.org and stats.cruxcoach.org, including the planned app update event, is the maintainer of the non-commercial CruxCoach open-source project. Privacy enquiries can be sent to press@cruxcoach.org.

We use the website counters solely to understand which public pages are loaded, count activations of the two installation options, and maintain and improve the site and its download paths. Starting with the planned app release 0.2.2, the verified-update counter will additionally show aggregate updater distribution by target version and source so we can monitor its uptake and reliability. To the extent that receiving a request involves personal data, the legal basis is Article 6(1)(f) GDPR. Our legitimate interests are the privacy-preserving operation, maintenance, and improvement of this non-commercial open-source project and its download distribution. We do not use the data for advertising, profiling, recognising people, or counting unique visitors.

Analytics hosting, recipients and retention

The endpoint at stats.cruxcoach.org runs on infrastructure provided by Hetzner Online GmbH at the selected Helsinki, Finland location. Hetzner provides the hosting infrastructure and necessarily handles connection data while delivering network traffic. The collector is not placed behind an analytics CDN or WAF. Access logging in the dedicated reverse proxy and collector is disabled; our analytics stack retains no raw request, IP address, user agent, referrer, or exact timestamp. The APK selector separately caches only public release metadata and the currently chosen source, never visitor data.

Accepted requests are folded immediately into UTC-day counters. Those aggregates are exported to metrics_anonymous.csv and published in the public cruxcoach-dlstats repository. They are retained without a fixed deletion period, including in repository history. They contain no raw event, identifier, or information that lets our analytics system identify a person or link separate events.

Your choices and rights

Providing a website analytics event is not a statutory or contractual requirement. DNT or GPC suppresses website analytics without preventing the APK selector from redirecting. Blocking or failing a separate page-view or Zapstore-click counter does not affect navigation or downloading. The direct-APK selector is itself the functional download route; if that hostname is blocked or unavailable, it cannot choose and redirect to a source. Where the GDPR applies and the statutory conditions are met, you have rights of access, rectification, erasure, and restriction; data portability applies only where its separate legal conditions are met. Because no raw event or identifier is retained, after aggregation we normally cannot retrieve or isolate a visit or click relating to you. You may lodge a complaint with a supervisory authority, particularly in the Member State of your habitual residence, place of work, or the alleged infringement. There is no automated decision-making or profiling.

Right to object: You may object at any time, on grounds relating to your particular situation, to processing based on Article 6(1)(f) GDPR. You can contact us at press@cruxcoach.org; enabling DNT or GPC also suppresses website analytics requests technically.

Map tiles on /boards/

The map background on /boards/ is rendered from OpenStreetMap raster tiles served by tile.openstreetmap.org. While you are viewing that page, your browser sends requests directly to that host for every tile it needs to draw. Those requests are subject to OpenStreetMap's tile-usage policy and the OpenStreetMap Foundation's privacy policy. We have no insight into those requests; the only way to avoid them is to not load the map.

What Codeberg does

This site is hosted by Codeberg e.V. on Codeberg Pages. Codeberg processes technical request metadata, including the IP address, browser and operating system, date and time, requested URL, and referrer, to provide the platform and prevent misuse. If technical metadata is written to logs, Codeberg says the IP address is truncated and the metadata is retained for no more than seven days unless German law requires otherwise. We have no access to it. See Codeberg's privacy policy for details.

External links

There is no visitor-side or pre-click APK availability probe. The first-party server checks source health without visitor data. After you activate the direct-APK button, your browser first contacts stats.cruxcoach.org and is then redirected to the selected Codeberg or Zapstore source; only then does it connect to that provider. Other external links connect directly when clicked. Those services have their own privacy practices, which apply at the moment you connect:

  • Codeberg: codeberg.org/Codeberg/org/PrivacyPolicy.md
  • Zapstore: zapstore.dev/docs/faq
  • Mastodon (mastodon.social): mastodon.social/privacy-policy
  • Primal: primal.net

Email

The address press@cruxcoach.org is forwarded to the project maintainer. Email content and routing metadata pass through the operator that provides the forwarding service. We retain no separate copy beyond what's needed to reply, and we don't use the address for any newsletter, marketing, or unsolicited contact.

The CruxCoach Android app

This page primarily describes cruxcoach.org. The CruxCoach Android app keeps personal data local by default, has no user tracking or accounts, and offers opt-in encrypted backup. Current release 0.2.1 sends no app-analytics event. The upcoming 0.2.2 release will add one aggregate increment after the optional in-app updater fully downloads an update and verifies its SHA-256 and signing certificate. The event will contain only the target version and source (codeberg/zapstore), never a device, installation, account, Nostr, advertising, or user identifier. A failed counter request will never affect update checking or installation. It will count verified updater downloads, not people or successful installs. The app's complete network behaviour is described on the main page and in the source repository at codeberg.org/CruxCoach/CruxCoach.
